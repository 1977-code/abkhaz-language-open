# Техническая дорожная карта

## До MVP

| Этап | Результат |
|---|---|
| T0 | статическая витрина проекта |
| T1 | схема данных уроков, слов, аудио, упражнений |
| T2 | локальный каталог уроков в Markdown/YAML |
| T3 | сайт-ридер: главы, аудио, словарь |
| T4 | интерактивные упражнения без аккаунтов |
| T5 | поиск по урокам и словарю |

## После MVP

| Этап | Результат |
|---|---|
| T6 | личный прогресс |
| T7 | интервальные карточки |
| T8 | корпусный поиск |
| T9 | pronunciation tools |
| T10 | AI-тьютор controlled mode |

## Выбор стека

Для первого сайта достаточно:

- статический генератор или простое React/Vite приложение;
- Markdown/YAML как исходники;
- JSON-экспорт для упражнений;
- отдельная папка аудио;
- строгий `rights_status` у каждого объекта.

## Требования к CMS/данным

- Не публиковать объект без `rights_status`.
- Не публиковать абхазскую фразу без `native_review=true`.
- Не публиковать грамматическое объяснение без `scientific_review=true`.
- Каждый аудиофайл связан с `speaker_release_id`.
