# Частотность ABNC v1

Дата извлечения: 11 июня 2026 года.

## Результат

- источник: [Abkhaz National Corpus](https://clarino.uib.no/abnc), `SRC-010`;
- лицензия корпуса: CLARIN PUB / CC0;
- объём запроса: 10 782 836 совпадений;
- уникальные значения атрибута `lemma`: 64 307;
- опубликованный агрегат: `data/freq_abnc_lemma_v1.csv`, top-5000;
- рабочая очередь: `data/lexical_review_queue_v1.csv`, первые 1500 кандидатов на лексикографическую проверку.

Выгрузка получена из публичного интерфейса Corpuscle: запрос `*` по всему корпусу, атрибут `lemma`, сортировка по частоте, регистр игнорируется, счётчики включены. Тексты, предложения и конкордансы в репозиторий не копировались.

## Воспроизведение

```bash
python3 06_corpus/scripts/fetch_abnc_frequency.py \
  data/freq_abnc_lemma_v1.csv --attribute lemma --limit 5000

python3 06_corpus/scripts/build_lexical_review_queue.py \
  data/freq_abnc_lemma_v1.csv data/lexical_review_queue_v1.csv \
  --candidate-limit 1500
```

Загрузчик сам создаёт публичную сессию ABNC, запускает запрос, ждёт его завершения и скачивает агрегированную TSV-выгрузку. Если интерфейс Corpuscle изменится, скрипт завершится ошибкой вместо создания частичного файла.

## Как читать данные

`corpus_lemma` — значение морфологической разметки ABNC, а не готовая форма для карточки ученика. В данных встречаются:

- морфемная сегментация дефисами и точками;
- знаки ударения;
- пунктуация и `??` для неразобранных токенов;
- имена собственные и формы с прописной буквы;
- возможные ошибки автоматического анализа.

Автоматический фильтр удаляет из кандидатной очереди только очевидный шум: строки без кириллической леммы, цифры, латиницу и маркеры неизвестного разбора. Он не назначает перевод, уровень или словарную форму.

## v2: POS-выгрузка (29.06.2026)

P. Meurer прислал частотную выгрузку с частью речи: `word-lemma-pos.tsv` (52 МБ, вне git; архив `06_corpus/word-lemma-pos.tsv.zip`, тоже вне git). Формат: `словоформа ⇥ часть речи ⇥ лемма ⇥ частота`, 1 223 404 строки-словоформы, ~7,24 млн токенов всего.

Обработка: `06_corpus/scripts/build_abnc_pos_v2.py` → **`data/freq_abnc_lemma_pos_v2.csv`** (top-6000): `rank, lemma, pos, freq, per_million, n_wordforms`. Агрегат по лемме (сумма по словоформам), доминирующая часть речи, число словоформ. Чистка как в v1.1 (только кириллические леммы без русских вкраплений). После чистки: ~6,49 млн токенов, 755 911 уникальных кириллических лемм.

Набор частеречных меток: V (глагол, доминирует — полисинтетика), Noun, Adj, VN (отглагольное имя), Adv, PP (послелог), Pron, Num, Cj, Prt, Interj, NH, Prop, Abbrev.

Известные особенности (для лексикографа):
- леммы сегментированы (дефисы, префиксы `с-/ҳ-/а-`, ударения) — это разметка ABNC, не словарная форма;
- регистровые дубли (`уи`/`Уи`, `аха`/`Аха`, `сара`/`Сара`) — заглавные начала предложений как отдельные леммы; можно слить регистронезависимо в v2.1;
- Meurer предупредил: при неоднозначном разборе пары (словоформа, лемма, часть речи) считаются с весом 1/nm; он планирует улучшить индексацию для корректной привязки леммы к части речи. Поэтому данные — черновые, обязательна проверка лексикографом и носителем.

## Контроль качества

До включения в учебный словарь каждая строка должна пройти:

1. проверку словарной формы и морфологического разбора лексикографом;
2. сверку перевода и регистра;
3. сопоставление с разговорным слоем Common Voice и сценариями глав;
4. проверку носителем;
5. разметку `A0`, `A1` или `A2`.

Частотность определяет порядок просмотра, но не заменяет методический отбор.
