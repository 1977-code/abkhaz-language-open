# Open data workflow

## Цель

Разделить данные, которые можно публиковать, и данные, которые можно использовать только внутри исследования.

## Категории

| Категория | Публиковать? | Примеры |
|---|---|---|
| `own-original` | да | тексты курса, упражнения |
| `public-open` | да, по условиям | CLARINO, Common Voice metadata |
| `licensed-third-party` | по договору | партнерские клипы |
| `research-only` | нет | PDF без разрешения |
| `permission-pending` | нет | материалы в ожидании ответа |

## Перед публикацией датасета

- Проверить каждый `source_id`.
- Удалить `research-only`.
- Удалить персональные данные.
- Проверить лицензии аудио.
- Добавить README.
- Добавить LICENSE.
- Добавить attribution.
- Добавить changelog.

## Перед использованием в AI

- Только `own-original`, `public-open`, `licensed-third-party`.
- Проверить, разрешает ли лицензия машинную обработку.
- Для голосов проверить speaker release.
- Сохранить список исключенных источников.
